## 一、功能概述

动态旅程回放是一个**电影般的动画回放功能**，让用户可以重温已归档的旅行路线。用户在全屏地图背景下，以第一视角体验沿旅行路线的平滑移动，每个节点停留时展示对应的照片和文字笔记。

### 用户故事

> 作为用户，我希望能够像看电影一样回放我的旅行路线，在地图上看到我的移动轨迹，并在每个重要地点看到我拍摄的照片和记录的文字，这样我就可以重温当时的美好回忆。

```

    ↓                  ↓                  ↓                  ↓

```

## 三、数据结构设计

### 3.1 回放数据模型

```typescript
/**
 * 回放路线 - 完整数据模型
 * 未来由路线生成器 (RouteGenerator) 从 MemoryNode[] 转换生成
 */
export interface ReplayRoute {
  id: string
  name: string
  nodes: ReplayNode[]      // 有序节点列表（按时间/用户定义顺序）
  totalDistance: number    // 总距离 (km)
  duration: number         // 总时长 (小时)
}

/**
 * 回放节点 - 单个记忆点
 */
export interface ReplayNode {
  id: string
  position: {              // 地理坐标
    latitude: number
    longitude: number
  }
  photoUri: string         // 照片资源路径
  title: string            // 地点名称
  note: string             // 完整文字笔记
  noteShort: string        // 缩略文字 (最多 20 字，带省略号)
  timestamp: number        // 小时级时间戳 (毫秒)
  order: number            // 在路线中的顺序
}

/**
 * 路线生成器接口
 * 未来由队友实现，将 MemoryNode[] 转换为 ReplayRoute
 */
export interface RouteGenerator {
  /**
   * 从记忆节点生成回放路线
   * @param nodes 记忆节点数组（来自 MemoryNodeRepository）
   * @returns 回放路线
   */
  generateFromNodes(nodes: MemoryNode[]): ReplayRoute

  /**
   * 计算节点间距离
   */
  calculateDistance(pos1: LatLng, pos2: LatLng): number
}
```

## 五、核心动画逻辑

### 5.1 状态机

```typescript
enum ReplayState {
  IDLE,        // 初始状态
  PLAYING,     // 播放中
  PAUSED,      // 已暂停
  COMPLETED    // 播放完成
}

@State currentState: ReplayState = ReplayState.IDLE
@State currentNodeIndex: number = 0
@State isCardVisible: boolean = true
@State progress: number = 0  // 0-100，用于进度条
```

### 5.2 播放逻辑伪代码

```typescript
class TripReplayPage {
  private playTimer: number = -1
  private readonly STAY_DURATION = 5000  // 5 秒停留
  private readonly MOVE_DURATION = 1500  // 1.5 秒移动

  // 开始/切换播放
  togglePlay(): void {
    if (this.isPlaying) {
      this.pause()
    } else {
      this.play()
    }
  }

  play(): void {
    this.isPlaying = true
    this.playSequence()
  }

  pause(): void {
    this.isPlaying = false
    clearTimeout(this.playTimer)
  }

  // 播放序列（递归）
  private playSequence(): void {
    if (!this.isPlaying || this.currentNodeIndex >= this.nodes.length) {
      return
    }

    // 1. 显示当前节点卡片
    this.showCard(this.currentNodeIndex)

    // 2. 设置定时器：5 秒后移动到下一节点
    this.playTimer = setTimeout(() => {
      if (!this.isPlaying) return

      const nextIndex = this.currentNodeIndex + 1
      if (nextIndex >= this.nodes.length) {
        // 播放完成
        this.isPlaying = false
        return
      }

      // 3. 移动到下一节点
      this.moveToNode(nextIndex)

      // 4. 递归调用，继续播放序列
      this.playSequence()
    }, this.STAY_DURATION)
  }

  // 移动到指定节点
  moveToNode(index: number): void {
    this.currentNodeIndex = index

    // 1. 隐藏卡片（移动过程中不显示）
    this.isCardVisible = false

    // 2. 相机沿曲线移动到目标节点（1.5 秒）
    const targetNode = this.nodes[index]
    this.animateCameraTo(targetNode.position, this.MOVE_DURATION, () => {
      // 动画完成回调
      this.isCardVisible = true
      this.progress = (index / (this.nodes.length - 1)) * 100
    })
  }

  // 相机动画（沿道路曲线）
  private animateCameraTo(
    target: LatLng,
    duration: number,
    onComplete?: () => void
  ): void {
    // 使用 MapKit 的 animateCamera API
    // 如果支持 waypoints，添加中间点实现曲线效果
    const cameraUpdate = map.newCameraPosition({
      target: target,
      zoom: 15,
      bearing: this.calculateBearing(this.currentPosition, target),
      tilt: 45  // 45 度倾斜，增加动感
    })

    this.mapController?.animateCamera(cameraUpdate, duration, onComplete)
  }

  // 计算两点之间的方位角（用于相机旋转）
  private calculateBearing(from: LatLng, to: LatLng): number {
    const dLon = (to.longitude - from.longitude) * Math.PI / 180
    const lat1 = from.latitude * Math.PI / 180
    const lat2 = to.latitude * Math.PI / 180
    const y = Math.sin(dLon) * Math.cos(lat2)
    const x = Math.cos(lat1) * Math.sin(lat2) -
              Math.sin(lat1) * Math.cos(lat2) * Math.cos(dLon)
    const bearing = Math.atan2(y, x) * 180 / Math.PI
    return (bearing + 360) % 360
  }
}
```

### 5.3 进度条交互

```typescript
// 离散步进进度条
Slider({
  value: this.progress,
  min: 0,
  max: 100,
  steps: this.nodes.length - 1  // 分成 N-1 段
})
.onChange((value: number) => {
  // 计算最近的节点索引
  const index = Math.round((value / 100) * (this.nodes.length - 1))
  this.jumpToNode(index)
})

jumpToNode(index: number): void {
  this.pause()  // 拖动进度条时暂停
  this.currentNodeIndex = index
  this.flyToNode(index)  // 直接跳转，不播放中间动画
  this.progress = (index / (this.nodes.length - 1)) * 100
}
```

## 八、技术风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| MapKit 不支持曲线动画 | 相机只能直线移动 | 降级为直线 lerp，通过调整 bearing 和 tilt 增加动感 |
| MapKit animateCamera 不支持回调 | 无法精确控制动画完成时机 | 使用 setTimeout 估算，或轮询相机位置 |
| 照片加载慢导致卡顿 | 动画过程中照片闪烁 | 预加载所有照片到内存，使用 ImageCache |
| 内存占用过高 | 大图导致 OOM | 使用缩略图，原图仅在展开时加载 |

## 九、验收标准 (UAT)

| 编号 | 测试场景 | 预期结果 |
|------|----------|----------|
| UAT-1 | 首次进入回放页面 | 自动定位到第一个节点，显示照片卡片 |
| UAT-2 | 点击播放按钮 | 5 秒后自动移动到下一节点，相机平滑移动 |
| UAT-3 | 移动过程中 | 卡片隐藏，相机沿曲线移动，耗时约 1.5 秒 |
| UAT-4 | 到达新节点 | 卡片显示新节点的照片和文字 |
| UAT-5 | 点击暂停 | 动画停止，定时器清除 |
| UAT-6 | 拖动进度条 | 暂停播放，跳转到对应节点，卡片更新 |
| UAT-7 | 点击照片卡片 | 弹出底部半屏覆盖层，显示大图和完整文字 |
| UAT-8 | 点击覆盖层遮罩 | 覆盖层关闭，返回回放页面 |
| UAT-9 | 播放完成 | 停在最后一个节点，播放按钮恢复为 ▶ |
| UAT-10 | 点击上一节点/下一节点 | 正确跳转，进度条更新 |
