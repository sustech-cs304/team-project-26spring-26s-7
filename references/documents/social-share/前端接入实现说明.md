### 怎么真机测试

1. **后端隧道域名**：`API_BASE_URL` 当前指向 `https://halloween-folder-unique-destination.trycloudflare.com`。隧道进程不死域名稳定；**重启 cloudflared 会换域名**，那时同步改 `ApiEndpoints.ets` 即可。

2. **跑通最小路径**：
   - DevEco Studio 打开 `frontend/`，连真机或模拟器
   - 在任意页面里跳到 `pages/SharePage`，可不带参数（会用 demo trip）：
     ```ts
     router.pushUrl({ url: 'feature/social-share/pages/SharePage' })
     ```
   - 进入 SharePage 后自动看到「正在生成签名链接…」，约 1–3 秒后展示真实 URL
   - 点「复制」按钮 → URL 进剪贴板 → 浏览器粘贴打开 → 看到服务端渲染的 H5 页面（含两张 demo 照片）

3. **网络权限**：app 需要 `ohos.permission.INTERNET`，应该已经在 `module.json5` 里。如果没有：
   ```json
   "requestPermissions": [
     { "name": "ohos.permission.INTERNET" }
   ]
   ```

### 把 demo 换成真实 RDB 数据（v0.2 待办）

把 `SharePage.publishShare()` 的两段 demo 代码换成：

```ts
// 1. 用 RdbDataService 取 trip + nodes
const rdb = await RdbDataService.getInstance(ctx)  // 看 common/service/RdbDataService.ets
const trip = await rdb.getTravelById(this.currentTripId)
const nodes = await rdb.getNodesByTravelId(this.currentTripId)
if (!trip || !nodes.length) {
  this.errorMsg = '路线不存在'
  this.isLoading = false
  return
}

// 2. 构造 SharePublishRequest
const req: SharePublishRequest = {
  tripId: trip.id,
  tripName: trip.name,
  totalDistance: trip.totalDistance,
  coverIndex: 0,
  createdAt: trip.createdAt,
  nodes: nodes.map<SharePublishNode>((n, idx) => ({
    id: n.id,
    title: n.title,
    content: n.content,
    poiName: n.poiName,
    photoCount: n.photos.length,    // 必须等于实际能上传的照片数
    mood: n.mood,
    tags: n.tags,
    visitedAt: n.visitedAt || n.createdAt,
    nodeOrder: n.nodeOrder ?? idx,
    latitude: n.latitude,
    longitude: n.longitude,
  })),
}

// 3. 把所有节点的照片路径扁平化、过滤、归一
const flatPaths: string[] = []
for (const n of nodes) {
  for (const p of n.photos) flatPaths.push(p)
}
const photos = SharePhotoHelper.toSharePhotos(flatPaths)

// 4. ⚠️ 校正 photoCount —— SharePhotoHelper 会过滤掉云端引用 / 缺失文件，
//    所以可上传数 < photos.length。要按实际"幸存"的照片数重算每节点 count。
//    最简单做法：对每个节点重新扫一遍它自己的 photos[]。
let cursor = 0
for (let i = 0; i < req.nodes.length; i++) {
  const survived = SharePhotoHelper.toSharePhotos(nodes[i].photos)
  req.nodes[i].photoCount = survived.length
}
const allPhotos: SharePhoto[] = []
for (const n of nodes) {
  const survived = SharePhotoHelper.toSharePhotos(n.photos)
  for (const sp of survived) allPhotos.push(sp)
}

// 5. 调 publish
const result = await this.shareService.publish(req, allPhotos, this.expireDays * 24)
```

### 已知限制（按时间紧迫程度排）

1. **systemShare 没接**：「分享到微信 / 微博」按钮当前是「复制 URL + 文案到剪贴板，让用户去 app 里粘贴」。后续接 `@kit.ShareKit` 的 `systemShare` 直接拉系统分享面板。
2. **隧道域名会变**：cloudflared 重启时换名。生产前换 AGC Hosting + 自定义域名。
3. **expireDays 是页面 state**：固定 7 天。可加一个 1/7/30 的选择器（后端已支持 `expiryHours ∈ [1, 720]`）。
4. **二维码没生成**：QRCodeShare.ets 还是占位实现。等接好真实 URL 后用 `@kit.NotificationKit` 或第三方库渲染 QR。
5. **错误展示比较糙**：直接把 `errorMsg` 当文本显示，包括 `errorDetail`。可以根据 errorCode 做更友好的文案映射（如 40004 → "照片太大，请压缩后再试"）。

### 与契约的对齐细节

| 契约条款 | 前端实现 |
|---|---|
| §4.1 multipart `tripData` text | `MultipartFormData.addField('tripData', JSON.stringify(trip))` |
| §4.1 multipart `photo_N` file | `MultipartFormData.addFile('photo_${i}', filename, path, contentType)` |
| §4.1 `expiryHours` text 默认 168 | `SharePage.expireDays * 24`，默认 `expireDays=7` |
| §4.2 `tripData.nodes[].photoCount` | 客户端先校验 sum；不匹配直接拒绝（不发请求） |
| §4.4 `code: 0` 成功 | `ShareService` 在 envelope 里检测 `code === 0 && data` 才视为成功 |
| §4.5 各错误码透传 | `ShareCallResult.errorCode + errorMessage + errorDetail` |
| §11 类型签名 | 一字不差，外加可选 `latitude/longitude`（对齐版文档加的字段） |
