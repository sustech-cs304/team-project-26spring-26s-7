TravelPin 项目审查报告

  一、项目概况

  TravelPin 是一款基于 HarmonyOS (ArkTS/ArkUI) 的旅行日记应用，核心功能包括地图打卡、路线回放、AI 文案生成、云同步和社交分享。后端使用 Python (Qwen-VL 视觉模型推理)。

  ---
  二、隐患 Bug 与安全问题

  [严重] 1. 签名证书和密钥文件提交到了 Git 仓库

  certificates/1.p12、test.cer、testDebug.p7b 都在版本控制中，且 .gitignore 未排除它们。build-profile.json5 中包含加密后的 storePassword 和 keyPassword。如果仓库公开或泄露，攻击者可以冒充你的应用签名。

  建议：立即将 certificates/ 加入 .gitignore，并用 git filter-branch 或 git filter-repo 从历史中清除。上架前必须轮换证书。

  [严重] 2. agconnect-services.json 包含 API 密钥

  华为 AGC 配置文件含有 client_secret、api_key 等加密凭证，已提交到仓库。

  建议：从 git 历史中清除，加入 .gitignore，上架前在华为 AGC 控制台轮换密钥。

  [严重] 3. bundleName 为测试名 com.ran.login_test

  app.json5 中包名还是 "com.ran.login_test"，vendor 也是 "example"。华为应用市场不会接受测试包名。

  [高危] 4. 路由参数未做空值校验

  多个页面直接 router.getParams() as RouterParam 进行类型断言，无 null 检查：
  - NodeEditPage.ets:43
  - TripDetailPage.ets:71
  - TripEditPage.ets:34
  - NodeDetailPage.ets:39

  如果参数为空会导致应用崩溃。

  [高危] 5. JSON.parse() 未包裹 try-catch

  云同步模块多处直接解析 JSON 字符串：
  - SyncManager.ets:94, 104
  - CloudSyncService.ets:128

  畸形数据会导致未捕获异常、应用崩溃。

  [高危] 6. 云操作（上传/下载）缺少超时机制

  CloudStorageService.ets 中的文件上传/下载使用 Promise 包装，但没有超时逻辑。弱网环境下 Promise 可能永远不 resolve，导致 UI 卡死。

  [中危] 7. fire-and-forget 异步调用吞没错误

  RdbDataService.ets 中多处 void this.uploadNodePhotosToCloud(...) 等调用没有 .catch()，错误被静默吞没，用户不知道云同步失败。

  [中危] 8. AppStorage 取值的类型安全问题

  多处 AppStorage.get('xxx') as boolean/number 直接断言，未考虑值不存在时返回 undefined 的情况，可能导致后续逻辑异常。

  [中危] 9. 敏感信息写入系统日志

  AuthService.ets 日志中输出了 token 长度、uid 等信息；SyncManager.ets 输出完整同步数据 JSON。发布版本应关闭或脱敏。

  [低危] 10. 网络监听器反注册时使用空回调

  EntryAbility.ets 的 onDestroy 中 this.netConnection?.unregister(() => {}) 用了空回调，无法感知反注册失败。
