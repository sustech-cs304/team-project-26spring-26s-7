## 问题 1：缺少《测绘资质证书》/ 互联网地图服务授权

### 审核意见摘录
> 您的应用内含有互联网地图服务相关内容...应当依法取得相应的互联网地图服务测绘资质，并在资质许可的范围内提交互联网地图服务。
> 修改建议：请补充提交《测绘资质证书》并在专业范围中注明互联网地图服务；**若接入第三方开放平台地图服务的，可提交双方合作协议（授权证明）**。

### 这是个人开发者必走的"授权证明"路径

本应用使用的是华为提供的 **Map Kit / Petal Maps**（导入 `@kit.MapKit`），不自建地图渲染服务，所以走审核员给的第二条路：提交**与华为的合作 / 授权证明**。冉明东本人无法以个人名义办理《测绘资质证书》（需要法人 + 行业资质评估），但作为 Map Kit 的下游集成方可以引用华为已经持有的测绘资质 + Map Kit 使用授权。


3. **`huawei-map-kit-license.pdf`** 获取办法（**已实测找到官方页面**）：

   华为提供官方《华为地图服务协议》页面：
   ```
   https://developer.huawei.com/consumer/cn/doc/app/64077474
   ```
   该页正文是华为面向集成方的《华为地图服务协议》，AGC 审核认这条路径作为"双方合作协议（授权证明）"的主体文件。

   **抓 PDF 步骤**（已实测验证）：
   - 用 **Chrome / Edge** 打开上面 URL（页面是 JS SPA 必须在浏览器里跑）
   - 页面顶部有一个明显的 **"点击下载华为地图服务协议"** 链接 → 点它直接下载官方排版的 PDF
   - PDF 文件名通常含日期，把它重命名为 `huawei-map-kit-license.pdf` 放进 zip

   > 优先用官方下载链而**不是** Ctrl+P 打印——官方 PDF 是华为排版好的正式文件，比浏览器打印出来的格式权威、可信度高。
   >
   > 该协议正文里明确写了 Map Kit 是 "华为及其附属公司运营和控制的一系列 SDKs、APIs 和/或地图服务的集合"，明确华为是地图服务提供方、我们是集成方——这就是 AGC 要看的"双方合作授权关系"的法律证据。

   **协议链说明**：本协议是补充协议，完整链是
   `《华为开发者服务协议》 → 《华为APIs使用协议》 → 本《华为地图服务协议》`。
   AGC 通常只看最后一份就够，如审核员临时索要上游两份，在华为开发者站
   "协议与准则" 区域同款 PDF 也能拿到。
