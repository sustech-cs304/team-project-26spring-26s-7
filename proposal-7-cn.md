# 项目提案：鸿蒙旅行日记应用

## Part I. 初步需求分析

### 项目概述

**TravelPin** 是一款基于地理位置的旅行日记应用。

用户可以将照片与配文精准钉在地图坐标上，一键生成动态旅行剪辑，并借助 AI 智能生成不同风格文案分享至社交平台。系统利用华为云空间与华为账号体系，实现同账号设备间的分布式数据管理，主打"离线优先"，在无网环境下也能流畅记录。同时，通过本地隐私清洗和加密签名分享，系统全方位保障用户隐私与数据安全。

### 1. 功能需求

本系统包含以下 5 项正交的功能特性：

**交互式地图：**
* 将照片、文字故事和心情标签以"记忆节点"的形式钉在真实地图上。
* 按时间范围、地理区域或自定义标签浏览和筛选节点。
* 节点密集时杜绝卡顿，支持聚合渲染，点击聚合簇可展开查看各条目详情。

**动态旅程回放：**
* 选择并组合记忆节点为一条旅行线路。
* 一键生成充满回忆的旅行剪辑。

**跨平台路线分享：**
* 为任意路线生成加密签名、限时有效的分享链接。
* 支持直接分享至微信或复制到系统剪贴板。
* 接收者无需安装 App 即可查看完整旅程。

**AI 辅助生成文案：**
* 用户可以输入简单草稿，系统结合旅途元信息生成社交媒体文案。
* 支持用户选择语气风格（如诗意、简洁、幽默）。

**多设备无缝同步：**
* 支持华为账号一键授权登录。
* 记忆节点、路线和草稿通过华为云空间在手机、平板、电脑之间自动同步。
* 在手机随手记录，在电脑大屏上继续编辑。

### 2. 非功能需求

**易用性**
* **交互效率：** 核心操作（添加/删除/修改节点、创建路线）应在主页面少数次点击内完成。
* **学习成本：** 系统界面设计与交互操作需保持简洁，帮助用户快速上手。
* **状态反馈：** 界面需对所有后台行为提供明确且非阻塞的视觉或触觉反馈，避免用户对系统状态产生困惑。
* **无障碍与适配：** 系统支持动态字体缩放，以服务不同年龄段的用户。

**性能**
* **地图渲染：** 动态聚合并选择性渲染节点与路线，确保在多数据点场景下也能保持流畅渲染。
* **AI 响应时间：** LLM 文案请求应在较短时间内输出首个 token（TTFB），并采取流式传输以保持交互的即时感。
* **应用启动：** 在标准 HarmonyOS 设备上，冷启动到可交互地图页面应在极短时间内完成。

**安全与隐私**
* **数据分级保护：** 遵循 HarmonyOS 数据分级标准，对用户数据进行敏感度标注。高敏感数据（如地理位置、身份凭证）采用更高级别的加密存储和访问控制，确保数据在任意设备间流转时均受到与风险等级匹配的防护。
* **安全分享：** 分享链接采用服务端签名机制，结合时间限制和防篡改校验；接收方需通过身份验证后方可访问内容，防止链接被未授权传播。
* **AI 内容合规：** 所有 AI 生成的文案在展示或存储前需经过内容安全过滤，确保符合法律法规及平台规范。

**可靠性与可用性**
* **离线优先：** 在完全无网络环境下，应用仍可正常使用基础功能（如创建节点、编辑草稿、浏览历史）；本地数据采用设备级加密存储，确保设备丢失时数据不被窃取。
* **数据恢复：** 提供容错机制，用户删除数据后在设定时间内可进行恢复；永久删除时执行安全擦除，确保数据无法通过技术手段恢复。
* **分布式同步：** 支持同华为账号设备间的可信同步，同步通道采用端到端加密，确保跨设备数据传输安全。

### 3. 技术需求

系统的运行环境、开发工具链和运行时技术栈如下：

* **客户端/前端（HarmonyOS App）：**
    * **语言与 UI 框架：** ArkTS 搭配 ArkUI 声明式 UI 框架，实现原生组件渲染。
    * **定位服务：** 采用系统级位置安全控件，在用户主动触发场景下获取位置信息，确保权限使用的透明可控。
    * **媒体访问：** 采用系统级 Photo Picker 机制，遵循最小权限原则——应用无需获取相册完整访问权限即可选择照片。
    * **本地持久化：** 采用 HarmonyOS 分级加密存储方案，敏感数据（如用户凭证）存储于可信执行环境（TEE），普通数据采用设备绑定加密存储于关系型数据库（RDB）。

* **后端与云端（华为云服务）：**
    * **用户认证：** 集成华为账号认证服务，支持双因素认证，确保账户安全。
    * **数据同步：** 利用华为云空间分布式数据管理能力，实现同账号设备间的自动同步；同步通道端到端加密，云端无法解密用户数据。
    * **媒体存储：** 照片和视频存储于用户华为云空间，应用通过授权访问，传输和存储过程均采用加密保护。
    * **网页分享门户：** 轻量级只读 Web 应用，分享的旅行路线需经过签名验证和有效期校验后方可访问。

* **AI 与计算服务：**
    *  **本地边缘 AI**： 利用 HarmonyOS 系统级视觉能力，在设备端完成基础图像处理（如场景分类、标签生成）——仅提取元数据，原始照片不出设备。复杂 AI 任务（如文案生成）由云端  LLM 处理，发送的仅为脱敏后的文本元数据。  
    * **云端 LLM：** 通过 AI 网关访问的远程大语言模型，用于文案生成和内容合规审核；发送至云端的仅为脱敏后的旅程元数据，不包含原始照片和用户身份信息。

* **开发与交付环境：**
    * **IDE：** DevEco Studio（HarmonyOS 官方 IDE），用于前端开发。
    * **版本控制：** Git + GitHub；基于分支的工作流，配合 Pull Request 代码审查。
    * **项目管理：** GitHub Projects 看板，用于 Sprint 规划和进度追踪。
    * **CI/CD：** GitHub Actions，用于每次 PR 的自动化代码检查、测试和构建验证。
    * **测试：** 核心逻辑模块的单元测试；同步和 API 层的集成测试。

### 4. 数据需求

* **需要哪些数据：**
    * **地理空间数据：** 实时 GPS 坐标（经度、纬度）、移动轨迹点、海拔、航向和时间戳——构成所有地图和路线功能的基础。
    * **媒体与内容数据：** 用户选择的照片和视频（通过 Photo Picker 访问）、手动输入的文字草稿、心情标签和旅行元数据（名称、日期范围、封面图片）。
    * **AI 上下文数据：** 从记忆节点聚合的旅程元数据——途经 POI 名称（通过逆地理编码获取）、总距离、旅行时长和用户提供的草稿文本——用作文案生成的输入。不向云端 LLM 发送任何照片。
    * **用户与同步数据：** 华为账号标识符、设备标识符、数据同步状态和冲突解决日志。

* **如何获取数据：**
    * **位置：** 在用户明确授权后，通过系统级位置安全控件获取；持续追踪仅在旅行记录会话期间激活。
    * **媒体：** 通过系统级 Photo Picker 访问，遵循最小权限原则——应用永远不会请求相册完整访问权限。
    * **POI 与地理编码：** 通过地图服务 API 进行逆地理编码，将坐标解析为可读的地点名称。
    * **AI 输入：** 旅程元数据（POI 列表、距离、时长）在本地聚合后，作为结构化文本发送至云端 LLM 网关；用户草稿文本为可选的附加输入。

* **数据处理与存储策略：**
    * **隐私保护**： 分享照片前于本地移除敏感元数据（如精确位置、设备信息），确保传播内容不包含用户隐私。AI 文案生成仅使用本地提取的场景标签，原始照片无需上传。
    * **分级存储：** 根据数据敏感度采用不同加密等级存储——高敏感数据（如身份凭证）采用设备锁屏密码绑定的加密方案，普通数据采用设备级加密。
    * **冷热分层：** 活跃旅行数据高频写入本地加密存储；历史数据在网络空闲时异步同步至华为云空间，同步过程采用端到端加密。
    * **数据生命周期：** 用户可永久删除任何节点或旅行，触发本地与云端数据的级联删除和安全擦除；分享链接在有效期后自动失效且不可恢复。
