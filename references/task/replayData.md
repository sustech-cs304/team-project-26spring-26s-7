  数据转换映射

  MemoryNode → ReplayNode 转换逻辑：

  MemoryNode.poiName      → ReplayNode.title      (地点名称)
  MemoryNode.content      → ReplayNode.note       (完整笔记)
  MemoryNode.content[:20] → ReplayNode.noteShort  (截断 20 字)
  MemoryNode.photos[0]    → ReplayNode.photoUri   (取第一张照片)
  MemoryNode.latitude     → ReplayNode.position.latitude
  MemoryNode.longitude    → ReplayNode.position.longitude
  MemoryNode.createdAt    → ReplayNode.timestamp
  数组索引 i              → ReplayNode.order

  结论：可以取消硬编码

  MockDataService 已经返回 MemoryNode 数据结构，只需在 TripReplayPage 中：
  1. 调用 dataService.getNodesByTravelId(tripId)
  2. 转换 MemoryNode[] → ReplayRoute
  3. 替换硬编码的 mockRoute

  问题 2：照片类型是什么情况？

  当前照片存储方式分析

  ┌─────────────────────────┬────────────────┬──────────────────────────┬─────────────────────────────────────────┐
  │          层级           │    照片类型    │           示例           │                显示方式                 │
  ├─────────────────────────┼────────────────┼──────────────────────────┼─────────────────────────────────────────┤
  │ 硬编码 (TripReplayPage) │ 应用内置资源   │ $r('app.media.photo_2')  │ ✅ 确保显示                             │
  ├─────────────────────────┼────────────────┼──────────────────────────┼─────────────────────────────────────────┤
  │ PhotoPickerUtil 返回    │ 系统相册 URI   │ "file:///data/media/..." │ ✅ 系统相册照片可显示                   │
  ├─────────────────────────┼────────────────┼──────────────────────────┼─────────────────────────────────────────┤
  │ MemoryNode.photos       │ 文件路径字符串 │ "file:///..." 或空数组   │ ⚠️ MockDataService.photos = [] 空数组！ │
  └─────────────────────────┴────────────────┴──────────────────────────┴─────────────────────────────────────────┘

  关键发现：MockDataService 照片数据是空的

  查看 MockDataService 第 157 行：
  photos: [],  // ← 空数组！没有模拟照片数据

  这意味着：
  - 当前状态：取消硬编码后，照片无法显示（photos 为空）
  - 需要补充：MockDataService 需要添加模拟照片路径

  照片显示的技术细节

  鸿蒙三种照片路径类型：

  1. 应用内置资源 $r('app.media.photo_2')
    - 存放在 entry/src/main/resources/base/media/
    - 确保显示 ✅
  2. 系统相册 URI "file:///data/service/el2/user/100/media/..."
    - PhotoPickerUtil 返回的真实路径
    - 需要权限访问，但在应用内可显示 ✅
  3. 应用沙盒路径 "file:///data/storage/el2/base/haps/entry/files/..."
    - 应用自己存储的照片
    - 可显示 ✅

  建议的操作方案

  方案：补充 MockDataService 的照片数据

  需要修改 MockDataService，让它返回带有模拟照片的 MemoryNode：

  // MockDataService.ets - 补充 photos 数组

  ]
