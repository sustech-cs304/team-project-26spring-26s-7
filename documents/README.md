# TravelPin Documents

This directory is organized by document purpose. Prefer adding new files to the
most specific existing directory instead of placing them at the root.

## Directory Map

- `design/`: architecture, UI design, sync design, security design, and diagrams.
- `deployment/`: CI/CD, Jenkins, GitHub Actions, Linux migration, and handoff notes.
- `features/`: feature-specific documents.
  - `features/ai-copy/`: AI copywriting feature notes.
  - `features/replay/`: replay enhancement plans, research, and media assets.
  - `features/social-share/`: social share requirements, API, and integration notes.
- `planning/`: task lists, sprint notes, and backend handoff items.
- `testing/`: test plans, coverage analysis, handoff notes, and course testing references.

## Maintenance Rules

- Keep current working documents in the relevant category directory.
- Keep large binary references close to the document set that uses them.
- Do not delete or overwrite old files during reorganizations; move them and update this index.
- Use lowercase English filenames for new project documents when practical. Existing Chinese or course-provided filenames can stay unchanged.
