# 技术需求文档：地图叙事 H5 海报分享系统 (增量实现版)

## 1. 项目概览

本项目旨在构建一个高兼容性的 H5 页面，通过自建服务器的限时 URL 分享。

- **第一阶段 (Phase 1)**：实现稳定的图文排版、地图背景、背景音乐及限时访问机制。
- **第二阶段 (Phase 2)**：在 Phase 1 的基础上，引入相机平移、缩放及图片的动态淡入淡出。

------

## 2. 第一阶段：核心功能实现 (MVP)

### 2.1 基础架构

- **前端**：Vite + HTML5 + CSS3 (使用 Flex/Grid 布局) + JavaScript。
- **后端**：Node.js (Express) 环境。
- **存储**：自建服务器本地磁盘，所有资源通过 Nginx 托管。

### 2.2 核心特性

1. **静态展示**：地图作为大背景（支持手势缩放或固定展示），海报图片以图文列表或瀑布流形式在地图上方展示。
2. **音频处理**：
   - 实现背景音乐自动/点击播放。
   - **关键兼容性**：针对微信环境，调用 `WeixinJSBridgeReady` 触发音频开启。
3. **限时 URL 机制**：
   - **签名算法**：后端根据 `ID + Expiry Time + Secret` 生成 HMAC 签名。
   - **校验拦截**：用户访问时，后端中间件校验签名合法性及是否过期。
4. **性能优化 (针对自建带宽)**：
   - **图片 WebP 化**：强制所有上传资源转换为 WebP。
   - **Nginx Gzip**：开启 `gzip_static on` 以减少传输体积。

------

## 3. 第二阶段：动态动画增强 (Incremental)

### 3.1 渲染范式迁移

- 从 DOM 布局迁移至 **Canvas (PixiJS)** 渲染，以支持高性能的动画。
- **相机控制**：使用 **GSAP** 驱动虚拟相机在地图坐标系（例如 4000x4000px）中移动。

### 3.2 动态交互

1. **路径叙事**：定义一系列关键帧坐标（Waypoints），相机按序平滑飞行。
2. **触发式淡入淡出**：图片不再一次性全部显示，而是当相机飞行至指定半径内时，触发 `alpha` 值从 0 到 1 的 GSAP 动画。

------

## 4. 详细技术参数说明

### 4.1 URL 结构建议

```
https://yourdomain.com/p/poster_id?t=1714280000&s=a7f2c9...
```

- `t`: Unix 时间戳（过期时间）。
- `s`: HMAC-SHA256 签名，防止篡改 `t`。

### 4.2 图片优化策略 (方案 A 必备)

| **策略**       | **描述**                                                     |
| -------------- | ------------------------------------------------------------ |
| **响应式加载** | 根据手机屏幕宽度（375px / 414px）分发不同分辨率的图片。      |
| **分段请求**   | Phase 1 使用懒加载；Phase 2 使用 PixiJS 资源分段加载。       |
| **缓存策略**   | 设置 `Cache-Control: public, max-age=31536000`，确保同一用户二次进入不耗带宽。 |

## 5. 给 Claude Code 的分步实施指令

### 步骤 A：构建后端安全网关

> "请编写一个 Node.js (Express) 后端：
>
> 1. 实现一个 `/generate` 接口，输入资源 ID，输出带 HMAC 签名和过期时间的 URL。
> 2. 实现一个路由拦截器，校验访问请求中的签名。
> 3. 设置静态资源目录，并配置基础的强缓存响应头。"

### 步骤 B：构建 Phase 1 前端页面

> "请使用 Vite 搭建一个 H5 页面：
>
> 1. 采用移动端适配方案（Viewport / Rem）。
> 2. 背景为一个可滚动的地图（全屏背景图）。
> 3. 页面包含一个音乐播放按钮，并处理微信/iOS 的自动播放逻辑。
> 4. 实现一个图片列表，所有图片使用 `loading='lazy'`。"

### 步骤 C：升级至 Phase 2 动画系统 (在完成上述步骤后)

> "请在现有项目基础上增加一个 WebGL 层：
>
> 1. 引入 PixiJS，将原本的静态地图和图片绘制到 Canvas 上。
> 2. 使用 GSAP 编写一个 Timeline，让相机在三个预设的地图坐标点之间移动。
> 3. 实现当相机接近图片坐标时，该图片淡入显示的逻辑。"

## 6. 风险预警

- **带宽预警**：若 10 个人同时访问，每张图 200KB，10 张图，瞬时流量约 20MB。如果自建服务器带宽仅 5-10Mbps，加载会非常缓慢。
- **对策**：在 Phase 1 阶段，务必加入一个 **Loading 进度条**，告知用户资源正在加载，防止用户以为页面崩溃。
