# LLM 服务器 API 接口规范

本文档描述了客户端（HarmonyOS App）与 LLM 服务器之间的所有通信内容。  
服务器同事可根据此文档配置和调整后端服务。

**当前服务器地址**: `http://172.18.35.215:8000`  
**当前模型**: `Qwen-VL-Chat-Int4`  
**客户端超时**: 60 秒

---

## 一、当前使用的端点（核心）

### `POST /v1/chat/completions` — 纯文本生成

这是目前客户端**实际调用**的端点。客户端不再上传图片，所有图片分析在设备本地完成，只发送纯文本 prompt。

#### 请求格式

```
Content-Type: application/json
```

```json
{
  "model": "Qwen-VL-Chat-Int4",
  "messages": [
    {
      "role": "user",
      "content": "请你作为一名旅行博主，照片构图：横幅构图。拍摄时段：傍晚。拍摄地点是「西湖」。用户补充说明：今天天气很好。用诗意盎然、优美浪漫的风格，写一段旅行文案。长度要求：3-5句话，约50-80字，适当展开描写。要求：生动描绘旅行场景和氛围，富有感染力，严格遵守长度要求。"
    }
  ],
  "temperature": 0.9,
  "top_p": 0.85
}
```

#### 期望的响应格式（标准 OpenAI 兼容）

```json
{
  "choices": [
    {
      "message": {
        "content": "夕阳西下，西湖的水面被染成一片金红。微风拂过柳枝，远处雷峰塔的剪影倒映在波光粼粼的湖面上。漫步苏堤，时光仿佛在这一刻静止，只留下满眼的诗意与温柔。"
      }
    }
  ]
}
```

客户端从 `choices[0].message.content` 提取生成的文案。

**备用解析**: 如果响应体顶层有 `content` 字段（而非 `choices` 数组），客户端也能兼容提取。

## 二、Prompt 构建规则

客户端在本地组装 prompt 后发送给服务器。prompt 由以下部分动态拼接：

### 固定前缀
```
请你作为一名旅行博主，
```

### 图片元数据（端侧提取，可能为空）

客户端在设备本地从照片 EXIF 中提取的信息，如果存在则拼入 prompt：

| 字段 | 来源 | 示例 | prompt 中的格式 |
|------|------|------|----------------|
| 照片构图 | 图片宽高比 | 横幅/竖幅/方形 | `照片构图：横幅构图。` |
| 拍摄时段 | EXIF DateTimeOriginal | 清晨/上午/中午/下午/傍晚/夜晚/深夜 | `拍摄时段：傍晚。` |

> 注：如果 EXIF 不可用或照片无元数据，这些字段会被省略，prompt 中不会出现。

### 用户输入（可选）

| 字段 | 示例 | prompt 中的格式 |
|------|------|----------------|
| 拍摄地点 | 西湖 | `拍摄地点是「西湖」。` |
| 用户补充说明 | 今天玩得很开心 | `用户补充说明：今天玩得很开心。` |

> 这两个字段都是用户可选输入，可能为空。

### 风格（必选）

用户从 4 种风格中选择一种，prompt 中使用中文描述：

| 风格 key | prompt 中的描述 |
|----------|----------------|
| `poetic` | 诗意盎然、优美浪漫 |
| `factual` | 客观纪实、真实记录 |
| `casual` | 轻松随意、生活化 |
| `minimal` | 极简主义、言简意赅 |

格式: `用{风格描述}的风格，写一段旅行文案。`

### 长度要求（必选）

用户从 3 个等级中选择，prompt 中使用详细描述：

| 长度 key | prompt 中的描述 |
|----------|----------------|
| `short` | 1-2句话，不超过30字，精炼概括 |
| `medium` | 3-5句话，约50-80字，适当展开描写 |
| `long` | 6-10句话，约120-200字，详细描绘场景细节和感受 |

格式: `长度要求：{长度描述}。`

### 固定后缀
```
要求：生动描绘旅行场景和氛围，富有感染力，严格遵守长度要求。
```

## 三、完整 Prompt 示例

### 示例 1：有 EXIF + 有地点 + 有补充（最完整）

```
请你作为一名旅行博主，照片构图：横幅构图。拍摄时段：傍晚。拍摄地点是「西湖」。用户补充说明：今天天气很好，湖面很平静。用诗意盎然、优美浪漫的风格，写一段旅行文案。长度要求：3-5句话，约50-80字，适当展开描写。要求：生动描绘旅行场景和氛围，富有感染力，严格遵守长度要求。
```

### 示例 2：有地点，无 EXIF，无补充

```
请你作为一名旅行博主，拍摄地点是「故宫」。用客观纪实、真实记录的风格，写一段旅行文案。长度要求：1-2句话，不超过30字，精炼概括。要求：生动描绘旅行场景和氛围，富有感染力，严格遵守长度要求。
```

### 示例 3：最简模式（无 EXIF，无地点，无补充）

```
请你作为一名旅行博主，用极简主义、言简意赅的风格，写一段旅行文案。长度要求：6-10句话，约120-200字，详细描绘场景细节和感受。要求：生动描绘旅行场景和氛围，富有感染力，严格遵守长度要求。
```

## 四、请求参数说明

### 固定参数

| 参数 | 值 | 说明 |
|------|------|------|
| `model` | `"Qwen-VL-Chat-Int4"` | 模型名称，客户端硬编码 |
| `temperature` | `0.9` | 采样温度 |
| `top_p` | `0.85` | nucleus sampling |
| `messages[0].role` | `"user"` | 始终为 user 角色 |

### 关于 messages 数组

客户端始终只发送**一条** user message，不发送 system message。如果服务端需要 system prompt，请在服务端自行注入。

## 五、旧端点（已废弃，但代码仍保留）

### `POST /v1/chat/completions/image` — 图片上传生成

> **已不再由客户端主动调用**，但 `VisionLLMClient.askImage()` 方法仍保留在代码中。

#### 请求格式
```
Content-Type: multipart/form-data
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `image` | file | 图片文件（application/octet-stream） |
| `model_name` | text | `"Qwen-VL-Chat-Int4"` |
| `question` | text | 提问文本 |
| `temperature` | text | `"1.0"` |
| `top_p` | text | `"0.8"` |

#### 响应格式

同 `/v1/chat/completions`，从 `choices[0].message.content` 提取。

## 六、健康检查端点

### `GET /health`

客户端有 `testConnection()` 方法调用此端点检测服务器是否在线。

期望响应: HTTP 200（响应体内容不重要）

## 七、服务端需确认/修改的事项

### 必须确认

1. **`/v1/chat/completions` 端点是否可用？** — 这是客户端现在实际调用的端点（纯文本，标准 OpenAI 兼容格式）。如果当前 Qwen-VL 服务只有 `/v1/chat/completions/image` 而没有纯文本端点，需要新增。

### 建议优化

2. **HTTPS** — 当前使用明文 HTTP，上架应用市场要求 HTTPS。建议配置 TLS 证书或在前面加 Nginx 反向代理。
3. **域名** — 当前硬编码内网 IP `172.18.35.215`，建议绑定域名。客户端配置在 `frontend/entry/src/main/ets/common/api/ApiEndpoints.ets` 的 `VISION_LLM_BASE_URL` 常量。
4. **超时** — 客户端超时 60 秒，如果模型推理经常超过 60 秒，请告知以便调整。
5. **`/health` 端点** — 如果没有，建议加一个简单的健康检查端点。

### 未来可能新增的端点（当前为 mock）

客户端代码中还预留了以下端点，目前返回的是本地 mock 数据：

| 端点 | 方法 | 说明 | 状态 |
|------|------|------|------|
| `/api/v1/ai/generate` | POST | 基于文本元数据生成文案（无图片） | 未实现，使用 mock |
| `/api/v1/ai/moderate` | POST | 内容审核 | 未实现，使用 mock |
| `/api/v1/share/{travel_id}` | GET | 分享链接验证 | 未实现 |

## 八、修改客户端配置的位置

如果服务器地址、端口、路径发生变化，需要修改以下文件：

| 修改内容 | 文件路径 | 变量名 |
|----------|----------|--------|
| 服务器基础地址 | `frontend/entry/src/main/ets/common/api/ApiEndpoints.ets` | `VISION_LLM_BASE_URL` |
| AI 生成超时 | 同上 | `HttpTimeout.AI_GENERATE` |
| 模型名称 | `frontend/entry/src/main/ets/common/api/AiGatewayClient.ets` | `textRequest.modelName` |
| temperature / top_p | 同上 | `textRequest.temperature` / `textRequest.topP` |
| prompt 模板 | 同上 | `buildPrompt()` 方法 |
| 风格/长度映射 | 同上 | `STYLE_PROMPTS` / `LENGTH_PROMPTS` |
